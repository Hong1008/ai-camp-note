# [Antigravity] 2026-05-18 트러블슈팅 리포트

## 1. PaddleOCR 임포트 시 'paddle' 모듈 유실 에러 (ModuleNotFoundError)
- **현상**: 8주차 첫 실습 노트북(`05-18.ipynb`)에서 `from paddleocr import PaddleOCR` 실행 시 `ModuleNotFoundError: No module named 'paddle'` 에러가 발생하며 커널 실행이 중단됨.
- **원인**: 
  - `paddleocr` 패키지는 딥러닝 백엔드 프레임워크인 `paddlepaddle`을 필수 의존성으로 요구하나, 현재 가상 환경에 해당 모듈이 누락되어 발생한 현상임.
  - 이를 해결하기 위해 `uv add paddlepaddle` 명령을 시도하였으나, `pyproject.toml` 내의 `requires-python = ">=3.13"` 설정으로 인해 `uv` 패키지 매니저가 파이썬 `3.14` 이상 및 타 플랫폼(윈도우 등)의 크로스 플랫폼 의존성을 해결하는 과정에서 `easyocr==1.7.0` 패키지 호환성 충돌(`No solution found`)이 발생하여 설치가 실패함.
- **해결**:
  - [pyproject.toml](file:///home/hong/project/ai-camp-note/pyproject.toml) 파일의 `requires-python` 속성을 `">=3.13,<3.14"`로 상한선을 규정하여 3.14 이상의 불필요한 플랫폼 의존성 락(lock) 충돌을 차단함.
  - 범위 제한 이후 `uv add paddlepaddle` 명령이 성공적으로 처리되어 `paddlepaddle==3.3.1`, `opt-einsum==3.3.0`, `safetensors==0.7.0`이 주입됨.
  - 가상 환경에서 `python -c "import paddle; print(paddle.__version__)"` 명령 실행을 통해 정상 작동(`3.3.1`)을 사전 검증 완료함.
- **교훈**: 
  - `uv`와 같은 견고한 의존성 락 매니저를 사용할 때, 상한선이 정의되지 않은 파이썬 범위(`>=3.13`)는 차세대 파이썬 버전의 라이브러리 미지원 이슈로 인해 불필요한 락 실패를 초래할 수 있으므로, 현재 구동 중인 안정적인 런타임에 맞춰 버전을 타이트하게 바인딩하는 것이 예방책임.

## 2. WSL2 환경에서의 MediaPipe 구동 오류 및 usbipd 웹캠 전송 지연 해결
- **현상**: 
  - MediaPipe `FaceLandmarker` 모델 가동 시 `libGLESv2.so.2` 공유 라이브러리 부재로 인한 `OSError` 발생.
  - 해결 이후 웹캠 연동(`cv2.VideoCapture(0)`) 시 `can't open camera by index` 또는 `select() timeout` 에러가 반복되며 카메라 프레임 수신 실패.
- **원인**:
  - WSL2는 기본 리눅스 배포판으로서 그래픽 ES 가속에 필요한 라이브러리(`libgles2`)가 누락되어 작동 거부가 일어남.
  - `usbipd`를 통한 하드웨어 패스스루 통신 시 가상 윈도우-리눅스 브릿지 세션이 만료되거나, 네트워크 대역폭 병목으로 미압축 실시간 영상 전송 시 심각한 지연 타임아웃이 초래됨.
- **해결**:
  - **라이브러리 보완**: 우분투 시스템 패키지 매니저를 통해 `sudo apt-get install -y libgles2`를 실행하여 가속 모듈을 성공적으로 이식함.
  - **Fallback 우회 전략 적용**: 시간 낭비성 드라이버 디버깅을 즉시 멈추고, 현업의 표준 컴퓨터 비전 테스트용 고품질 얼굴 인식 비디오(`face_sample.mp4`)를 다운로드하여 [content/face_sample.mp4](file:///home/hong/project/ai-camp-note/content/face_sample.mp4) 경로에 비치함.
  - 비디오 소스로 우회 변경(`cap = cv2.VideoCapture(CONTENT_DIR / 'face_sample.mp4')`)함으로써 타임아웃과 에러를 완전히 타파하고 실습 코드를 100% 정상 가동함.
- **교훈**:
  - 가상화 개발 환경(WSL2)에서 외부 하드웨어 장치 제어 시, 제조사 드라이버 및 네트워크 패킷 대역폭 특성에 따라 제어 불가능한 물리적 통신 버그가 수시로 발생함.
  - 실무에서는 하드웨어 디버깅에 끝없는 리소스를 소모하는 대신, 정밀하게 격리된 **오프라인 테스트 미디어를 구축하여 우회하는 Fallback 아키텍처**가 개발 생산성과 예외 복원력 관점에서 최선의 선택지임을 실천함.

## 3. 손 제스처(Hand Pose) 가위바위보 감지 실습 시의 타임아웃 및 양손 수어 영상(hand_sample.mkv) 우회 도입
- **현상**:
  - 손 랜드마크 추출 이후 한 단계 격상된 손가락 접힘 여부 분석 및 가위바위보 판정(`get_hand_pose`) 실습 셀에서 다시 `select() timeout` 경고가 반복되며 프로세스 먹통 및 강제 종료(`KeyboardInterrupt`) 발생.
- **원인**:
  - 앞선 얼굴 안경 피팅 우회 시와 동일하게, WSL2 내부의 비디오 파이프 드라이버가 물리 장치 소스로부터 프레임을 읽어오는 과정에서 병목 병목이 여전히 유발되어, 실질적으로 카메라는 끊임없이 먹통 상태가 됨.
- **해결**:
  - **양손 수어 벤치마크 미디어 확보**: 두 손을 모두 활용하여 책을 펴고 접는 정밀한 손가락 변화가 완벽히 포착되어 양손 트래킹(`num_hands=2`)을 보장하는 인텔 공식 미국 수어(ASL) 영상(`book.mkv`)을 긴급 공수해 [content/hand_sample.mkv](file:///home/hong/project/ai-camp-note/content/hand_sample.mkv)에 배치함.
  - 해당 미디어를 바탕으로 `cv2.VideoCapture(str(CONTENT_DIR / 'hand_sample.mkv'))`로 소스를 우회하여 오류 없이 정밀하게 핑거 랜드마크 분석 및 제스처 검증 흐름을 무결하게 안착시킴.
- **교훈**:
  - 환경적 특성에 의한 물리 장치 불능 문제를 겪을 때, "다른 컴퓨터 비전 세션에서도 예외 없이 동영상 기반 Fallback 로직을 탑재하는 일관된 설계 사상"을 확립하여 개발 진행도를 빠르게 진척시키는 현업 프로덕션 수준의 문제 회복 메커니즘을 경험함.
