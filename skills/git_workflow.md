# Skill: 학습 중심 Git 워크플로우

본 문서는 실무 중심의 기술적 규격보다는 **'학습 단계와 성과 기록'**에 초점을 맞춘 Git 운영 방식을 정의합니다.

---

## 1. 브랜치 규칙 (Branching)

- **생성 시점**: 매일 학습을 시작할 때 생성합니다.
- **명명 규칙**: `daily/yyyy-mm-dd`
    - 예: `daily/2026-04-10`
- **목적**: 하루 동안의 모든 시도와 실습 내용을 격리된 환경에서 안전하게 수행합니다.

---

## 2. 커밋 규칙 (Commits)

- **시점**: 하나의 기능 구현이 완료되거나, 의미 있는 학습 단계를 마칠 때 수시로 수행합니다.
- **포맷**: `[학습 주제]: [작업 요약]`
    - **학습 주제**: 현재 연구 중인 API 명칭이나 기술 테마 (예: `API 호출 실습`, `데이터 검증 실습`)
    - **작업 요약**: 수행한 작업 내용을 1~2줄로 요약하여 기술합니다.
    - 예: `[API 호출 실습]: 네이버 검색어 트렌드 API 클라이언트 구현 및 명세서 작성`

---

## 3. 마무리 규칙 (Daily Wrap-up)

하루의 학습이 종료되면 다음의 순서로 작업을 마무리합니다.

1. **문서화**: 해당 주차의 `README.md`에 오늘의 실습 핵심 내용을 요약하여 추가합니다.
2. **검증**: 모든 코드가 정상적으로 동작하고 테스트를 통과하는지 확인합니다.
3. **병합(Merge)**: `main` 브랜치로 **Squash Merge**를 수행합니다.
    - 하루의 복잡한 커밋 내역을 하나의 깔끔한 성과물로 통합합니다.
    - **커밋 제목**: `[yyyy-mm-dd]: [오늘의 주요 학습 성과 요약]`
    - **커밋 본문**: `Detail Commits:` 항목을 만들어 그날의 모든 개별 커밋 메시지를 포함합니다.
4. **정리**: 사용이 완료된 `daily` 브랜치를 삭제합니다.
5. **원격 저장소 반영(Push)**: 로컬의 `main` 브랜치를 `origin`으로 푸시하여 모든 학습 성과를 원격지에 안전하게 보관합니다.
    - 이전 커밋을 수정(Amend)한 경우 `git push origin main --force`가 필요할 수 있습니다.

---

## 4. AI 에이전트 자동 커밋 활용

- 사용자님이 **"지금까지 한 거 기능별로 커밋해줘"**라고 요청할 수 있습니다.
- 저는 변동된 파일들을 분석하여 위 규칙(`[학습 주제]: [작업 요약]`)에 따라 논리적 단위로 분할하여 자동 커밋을 수행합니다.
