# Skill: 트러블슈팅 로그 관리 (Troubleshooting)

기술적 이슈와 해결 과정을 체계적으로 기록하여 지식의 자산화를 돕는 기술입니다.

---

## 1. 기록 원칙 (Principles)

1. **즉시 기록**: 에러 해결 직후, 맥락이 생생할 때 가장 먼저 기록합니다.
2. **검색 용이성**: 에러 메시지 원문을 포함하여 나중에 같은 에러 발생 시 검색이 가능하게 합니다.
3. **교훈 중심**: 단순히 고친 코드를 보는 것이 아니라, "왜" 그런 일이 발생했고 "어떻게" 예방할지(Skill 업데이트 등)에 집중합니다.

---

## 2. 저장 규칙 (Storage Rules)

- **경로**: `week{N}/troubleshooting/{yyyy-mm-dd}.md`
- **파일명**: 날짜 형식(`yyyy-mm-dd`)을 따릅니다.
- **폴더**: 해당 주차 폴더(`week3` 등) 하위에 `troubleshooting` 폴더를 생성하여 관리합니다.

---

## 3. 표준 포맷 (Standard Format)

```markdown
# [에이전트] [날짜] 트러블슈팅 리포트

## 1. [이슈 요약 제목]
- **현상**: 에러 메시지 원문 또는 잘못된 동작 설명
- **원인**: 문제의 근본적인 이유 (라이브러리 제약, 설정 오타 등)
- **해결**: 적용한 코드 변경 사항 또는 설정 값
- **교훈**: 이 작업에서 배운 핵심 원칙

---
(추가 이슈가 있을 경우 반복)
```

---

## 4. 활용 (Usage)

- 작업 완료 후 또는 세션 종료 전, 제가 사용자님께 **"오늘의 트러블슈팅을 정리해 드릴까요?"**라고 먼저 제안할 수 있습니다.
- 사용자님도 직접 기록을 요청하거나 내용을 보충하실 수 있습니다.
