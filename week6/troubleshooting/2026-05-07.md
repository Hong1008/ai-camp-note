# [Antigravity] 2026-05-07 트러블슈팅 리포트

## 1. 차원 축소 기법(PCA vs LDA vs SVD)의 성능 차이 분석
- **현상**: 동일한 2차원 축소임에도 기법별로 정확도 차이 발생 (LDA: 95.3% > PCA/SVD: 88%)
- **원인**: 
    - **PCA/SVD**: 타겟 라벨을 고려하지 않는 비지도학습으로, 데이터의 전체 분산은 잘 보존하지만 클래스 간 경계 정보를 일부 손실함.
    - **LDA**: 타겟 라벨을 활용하는 지도학습으로, 클래스 간 분리를 최대화하는 축을 찾아 분류 성능을 극대화함.
- **해결**: 분류가 목적인 프로젝트에서는 LDA를, 일반적인 데이터 압축 및 시각화가 목적일 때는 PCA를 선택하는 전략 수립.
- **교훈**: 모델의 목적(분류 vs 군집 vs 시각화)에 따라 적절한 차원 축소 알고리즘 선택이 필수적임.

## 2. TruncatedSVD와 PCA의 수학적 동일성 확인
- **현상**: TruncatedSVD를 적용한 결과가 PCA와 소수점 끝자리까지 동일하게 나타남.
- **원인**: Scikit-learn의 TruncatedSVD는 입력 데이터가 스케일링(Center화)되어 있을 경우 PCA와 수학적으로 동일한 행렬 분해 과정을 거침.
- **교훈**: 데이터가 희소 행렬(Sparse Matrix)이 아닐 때는 굳이 SVD를 고집할 필요가 없으나, 텍스트 데이터 등의 특수 상황을 위해 SVD의 원리를 이해해두는 것이 중요함.

## 3. 실무적 관점에서의 PCA 적용 한계 (신용카드 연체 예측 사례)
- **사례**: `BILL_AMT1~6` 변수를 PCA로 압축하여 결합한 결과, 정확도는 유지(81.67%)되었으나 실무적 리스크 발견.
- **한계 및 단점**:
    1. **해석력(Interpretability) 유실**: 개별 달의 청구 금액이 연체에 미치는 직접적인 영향도를 파악하기 어려움.
    2. **파이프라인 관리 포인트 증가**: Scaler, PCA 객체 등 관리해야 할 전처리 객체가 늘어남.
- **교훈**: 성능과 해석력 사이의 트레이드오프를 고려하여 PCA 도입을 결정해야 함.

## 4. 군집화 평가 지표의 상대성 (KMeans vs GMM)
- **현상**: 동일 조건에서 실루엣 계수가 KMeans(0.55)가 GMM(0.50)보다 높게 나타남.
- **분석**: 실루엣 계수는 유클리드 거리를 기반으로 하므로, 거리 기반인 KMeans에 유리함. 점수가 낮다고 GMM이 나쁜 것이 아니라, 군집의 경계가 확률적으로 형성되었음을 의미함.
- **교훈**: 군집화 평가는 단일 지표에 의존하기보다, 실제 타겟 분포와의 일치도나 비즈니스적 유의성을 함께 검토해야 함.

## 5. Yellowbrick `SilhouetteVisualizer` 타입 에러 해결
- **이슈**: `YellowbrickTypeError: The supplied model is not a clustering estimator` 발생.
- **원인**: Scikit-learn 버전 업데이트로 인해 모델의 내부 속성(`_estimator_type`)을 Yellowbrick이 인식하지 못하는 문제.
- **해결**: 시각화 도구 호출 시 `force_model=True` 파라미터를 추가하여 강제로 모델 타입을 지정함.
