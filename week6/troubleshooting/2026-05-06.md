# CRISP-DM 기반 HAR 프로젝트 요약 (2026-05-06)

오늘 진행한 인간 행동 인식(HAR) 모델 구축 실습을 CRISP-DM 방법론에 따라 정리한 리포트입니다.

## 1. CRISP-DM 단계별 실습 내용

### 1단계: Business Understanding (비즈니스 이해)
- **목표**: 스마트폰 가속도/자이로 센서 데이터를 기반으로 사용자의 6가지 활동(Walking, Laying 등)을 자동 분류하는 고성능 모델 개발.

### 2단계: Data Understanding (데이터 이해)
- **데이터 분석**: 561개의 방대한 피처 중 `tGravityAcc` 등 중력 가속도 관련 피처가 활동 분류에 결정적인 역할을 함을 확인.
- **시각화**: Boxplot을 통해 'Laying' 활동이 특정 피처 영역에서 타 활동과 완벽히 분리(Discriminatory Power)되는 물리적 특성을 시각적으로 검증.

### 3단계: Data Preparation (데이터 준비)
- **Label Encoding**: XGBoost 모델 학습을 위해 문자열 타겟(Activity)을 정수형으로 변환.
- **Feature Selection**: 전체 561개 변수 중 모델 중요도 상위 95개를 선별하여 데이터 경량화 및 노이즈 제거.

### 4단계: Modeling (모델링)
- **알고리즘**: Decision Tree에서 시작하여 앙상블 모델인 XGBoost로 고도화.
- **최적화**: Optuna 프레임워크를 도입하여 `n_estimators`, `learning_rate` 등 주요 하이퍼파라미터 자동 튜닝 수행.

### 5단계: Evaluation (평가)
- **검증 지표**: `matrix_c` 함수를 정의하여 Accuracy, F1-Score(Macro), Confusion Matrix를 통합 관리.
- **성능 비교**: '전체 피처 모델' 대비 '상위 95개 피처 모델'의 성능 차이를 비교 분석하여 효율적인 모델 선정.

### 6단계: Deployment (배포)
- **모델 저장**: `joblib` 라이브러리를 사용하여 최종 모델과 LabelEncoder를 `.pkl` 파일로 저장.
- **예측 시스템**: 새로운 센서 데이터 입력 시 활동명을 출력하는 예측 파이프라인 구축.

---

## 2. 주요 트러블슈팅: XGBoost 타겟 라벨 오류
- **문제**: XGBoost 학습 시 `ValueError: Invalid target type` 발생.
- **원인**: XGBoost 최신 버전은 타겟값이 문자열인 경우 자동으로 처리하지 못함.
- **해결**: `sklearn.preprocessing.LabelEncoder`를 사용하여 `y_train`, `y_test`를 정수형(0~5)으로 인코딩 후 학습 진행.
- **교훈**: 모델 라이브러리마다 허용되는 데이터 타입이 다르므로 전처리 단계에서 명시적인 타입 변환이 필수적임.

## 3. 학습 및 시사점
- **Feature Importance의 가치**: 모델이 왜 특정 데이터를 중요하게 보는지 시각적으로 이해(Explainable AI의 기초)하는 과정이 모델 신뢰도 향상에 필수적임.
- **자동화 튜닝**: Optuna와 같은 도구를 활용하면 수동 튜닝보다 훨씬 빠르고 체계적으로 최적의 성능에 도달할 수 있음.
