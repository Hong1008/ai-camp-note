NumPy와 Pandas 기초를 학습하며 발생한 초기 세팅 및 데이터 처리 이슈를 기록합니다.

## 1. 이미지 데이터 연산 시 오버플로우 (Overwrite)
- **현상**: 이미지 픽셀값에 숫자를 더했더니 사진의 일부가 갑자기 검게 변함.
- **원인**: 이미지 배열의 기본 타입인 `uint8`(0~255) 범위를 넘어가면 숫자가 다시 0부터 시작하는 오버플로우 발생 (예: 250 + 10 = 4).
- **해결**: `np.clip()`을 사용하여 계산 결과의 범위를 0~255로 강제 고정하거나, 연산 전 데이터 타입을 `float`으로 변환 후 다시 `uint8`로 복구.
- **교훈**: 행렬 데이터 연산 시에는 해당 데이터 타입의 허용 범위를 반드시 의식해야 함.

## 2. NumPy 슬라이싱과 파이썬 리스트 슬라이싱의 차이
- **현상**: 슬라이싱한 데이터를 수정했는데 원본 데이터까지 같이 수정됨.
- **원인**: 파이썬 리스트는 슬라이싱 시 '복사본'을 만들지만, NumPy는 메모리 효율을 위해 원본을 가리키는 **'뷰(View)'**를 반환함.
- **해결**: 원본을 유지하고 싶다면 `.copy()`를 명시적으로 호출해야 함.
- **교훈**: 넘파이는 대용량 데이터를 다루기 때문에 복사보다는 참조(View)를 기본으로 한다는 철학을 이해해야 함.

## 3. `ipykernel` 설치 및 개발 환경 분리
- **현상**: VS Code에서 주피터 노트북 커널이 선택되지 않음.
- **원인**: 현재 가상환경에 `ipykernel` 패키지가 설치되어 있지 않음.
- **해결**: `uv add --dev ipykernel` 명령어로 설치. 배포 환경과 개발 환경을 구분하기 위해 `--dev` 옵션 활용.
- **교훈**: 주피터 환경은 단순히 UI만 있다고 돌아가는 것이 아니라, 엔진 역할을 하는 커널 패키지가 환경별로 필요함.
