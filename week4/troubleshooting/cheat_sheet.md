# Week 4: 데이터 분석 (EDA) 핵심 요점 노트

4주차에 진행한 데이터 분석(EDA) 과정에서 반드시 기억해야 할 통계 검정 및 시각화 도구들을 정리한 퀵 레퍼런스입니다.

---

## 1. 단변량 분석 (Univariate Analysis)
변수 하나하나의 분포와 특성을 파악하는 단계입니다.

| 구분 | 변수 타입 | 분석 도구 (시각화) | 분석 도구 (통계량) | 주요 확인 사항 |
| :--- | :--- | :--- | :--- | :--- |
| **숫자형** | Numeric | `histplot`, `boxplot` | `describe()`, `skew()`, `kurt()` | 기초통계량, 분포 모양, 이상치 유무 |
| **범주형** | Categorical | `countplot` | `value_counts()` | 범주별 빈도수, 클래스 불균형(Imbalance) |

---

## 2. 이변량 분석 (Bivariate Analysis)
두 변수 간의 관계를 분석하여 가설을 검증하는 단계입니다.

| 분석 유형 (X → Y) | 시각화 도구 | 통계 검정 (P-value) | 핵심 지표 |
| :--- | :--- | :--- | :--- |
| **숫자 → 숫자** | `regplot`, `jointplot`, `scatter` | `spst.pearsonr()` | 상관계수(r) |
| **범주 → 숫자** | `barplot`, `boxplot` | 2집단: `ttest_ind()` / 3집단: `f_oneway()` | T-통계량, F-통계량 |
| **범주 → 범주** | `mosaic`, `countplot(hue)` | `spst.chi2_contingency()` | 카이제곱 통계량 |
| **숫자 → 범주** | `kdeplot(multiple='fill')` | (구간화 후) 카이제곱 검정 | 확률/비중 변화 |

---

## 3. 통계 검정 및 결과 해석 가이드

### 3.1. 검정 도구 결정 트리
1.  **숫자 vs 숫자**: 상관계수 분석
2.  **범주(2개) vs 숫자**: Independent T-test
3.  **범주(3개 이상) vs 숫자**: One-way ANOVA
4.  **범주 vs 범주**: Chi-square Test

### 3.2. P-value 해석 표준
*   **$P < 0.05$**: 통계적으로 유의미함 (관계가 있음)
*   **$P \ge 0.05$**: 통계적으로 유의미하지 않음 (관계가 없음)

---

## 4. 시각화 옵션 꿀팁 (Seaborn)

| 옵션 | 용도 | 비고 |
| :--- | :--- | :--- |
| `common_norm=False` | 집단별 인원 차이가 클 때 분포 모양 비교 | `kdeplot` 전용 |
| `multiple='fill'` | 특정 수치 지점에서의 **집단별 비중(확률)** 확인 | `kdeplot` 전용 (강력 추천) |
| `hue` | 범주형 변수를 기준으로 색상 구분 | 대부분의 Seaborn 함수 적용 가능 |
| `gap` | 모자이크 플롯에서 칸 사이 간격 조정 | `mosaic` 전용 |

## 5. 핵심 이론 및 데이터 전처리 (Theoretical Concepts)

### 5.1. 이상치 (Outliers)
데이터의 전체적인 패턴에서 크게 벗어난 값입니다.
*   **판단 기준**: 
    *   **IQR 방식**: $Q1 - 1.5 \times IQR$ 보다 작거나 $Q3 + 1.5 \times IQR$ 보다 큰 값.
    *   **Z-Score 방식**: 평균으로부터 $\pm 3$ 표준편차 이상 떨어진 값.
*   **처리 전략**: 
    *   단순 제거 (데이터 손실 주의)
    *   상한/하한값으로 대체 (Capping/Winsorizing)
    *   로그 변환 등을 통한 분포 완화

### 5.2. 다중공선성 (Multicollinearity)
독립 변수들끼리 서로 강한 상관관계를 가지는 현상입니다.
*   **문제점**: 모델의 회귀 계수가 불안정해지며, 개별 변수의 영향력을 정확히 평가하기 어려워짐.
*   **진단**: 상관 계수 행렬 확인(0.8↑), VIF 확인(10↑).
*   **해결**: 상관관계가 높은 변수 중 하나를 제거하거나 결합.

### 5.3. 상관관계 vs 인과관계 (Correlation vs Causation)
*   **주의**: 상관관계($r$)가 높다고 해서 반드시 원인과 결과 관계인 것은 아님.
*   **잠재 변수(Latent Variable)**: 두 변수 모두에 영향을 주는 제3의 변수가 있는지 항상 의심해봐야 함.

### 5.4. 심슨의 역설 (Simpson's Paradox)
*   **현상**: 집단별 추세가 합쳐졌을 때 반전되는 현상.
*   **대응**: 전체 데이터만 보지 말고, 주요 범주별로 쪼개어 보는 **세분화 분석**이 필수임.

### 5.5. P-value의 한계 (Statistical vs Practical Significance)
*   데이터가 너무 많으면 아주 미세한 차이도 유의미($P<0.05$)하게 나옴.
*   **판단**: 통계적 유의성뿐만 아니라, 실제 비즈니스에 임팩트가 있는 차이인지를 보는 **실무적 유의성**을 함께 판단해야 함.

## 6. 데이터 분석 프로세스 (Standard EDA)
1.  **가설 수립**: "X(예: 거리)에 따라 Y(예: 퇴사)가 달라질 것이다."
2.  **단변량 분석**: 각 변수의 상태 확인 (결측치, 이상치, 분포).
3.  **이변량 분석 (시각화)**: 차트를 통해 직관적인 관계 파악.
4.  **이변량 분석 (통계 검정)**: P-value를 통해 관계의 객관적 증명.
5.  **종합 및 순위 산정**: P-value 기준으로 핵심 변수(Key Drivers) 선정 및 비즈니스 인사이트 도출.

---
> [!TIP]
> 모든 분석의 끝은 **"그래서 비즈니스 관점에서 무엇을 해야 하는가?"**에 대한 대답이어야 합니다. 통계치는 그 대답을 뒷받침하는 강력한 증거일 뿐입니다.
