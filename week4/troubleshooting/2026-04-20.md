# [에이전트] 2026-04-20 트러블슈팅 리포트

다중 데이터 병합과 데이터 전처리 기법을 학습하며 발생한 기술적 의문점과 해결 과정을 기록합니다.

## 1. VS Code 포트 포워딩 폭주 현상 (42개 포트 감지)
- **현상**: VS Code 하단 [PORTS] 탭에 원하지 않는 포트 40여 개가 자동으로 포워딩됨.
- **원인**: VS Code의 `Automatic Port Forwarding` 기능이 주피터 커널 통신용 포트 및 언어 서버(LSP)의 내부 포트를 모두 서비스용으로 오인하여 로컬로 연결함.
- **해결**: [Close Unused Ports] 기능을 사용하여 현재 통신이 없는 포트 일괄 정리. 습관적으로 터미널 프로세스를 `Ctrl+C`로 종료하는 습관 권장.
- **교훈**: 원격 개발 환경에서는 에디터의 '지나친 친절함'이 환경을 어지럽힐 수 있으므로, 포트 자동 관리 설정을 숙지해야 함.

## 2. 가변수(Dummy Variable)의 함정: 다중공선성
- **현상**: 범주형 데이터를 모두 가변수로 만들었을 때 모델 성능이 불안정해질 위험 인지.
- **원인**: N개의 범주를 N개의 열로 만들면 한 열의 값을 다른 열들의 합으로 완벽히 설명할 수 있는 '완전공선성'이 발생함. (예: Male + Female = 1)
- **해결**: `pd.get_dummies(..., drop_first=True)` 옵션을 사용하여 기준이 되는 하나의 열을 삭제(N-1개 유지)함으로써 수치적 중복 제거.
- **교훈**: 통계 기반 모델링에서는 데이터의 '중복된 의미'가 수학적 오류를 일으킬 수 있음을 유의해야 함.

## 3. 병합 시 데이터 타입 변경 (Int → Float)
- **현상**: 정수형 데이터가 포함된 두 테이블을 `outer join`하면 정수가 실수(`1.0`)로 변함.
- **원인**: `outer join` 시 발생하는 빈 칸(`NaN`)은 NumPy/Pandas에서 **실수(Float)** 타입임. 판다스 컬럼은 단일 타입을 유지해야 하므로 전체가 실수형으로 업캐스팅됨.
- **해결**: 데이터 타입 유지가 중요하다면 `astype('Int64')`(Nullable Integer)를 사용하거나 결측치를 먼저 채운 뒤 변환.
- **교훈**: 결측치(`NaN`)는 물리적으로 '실수'로 취급된다는 점을 데이터 설계 시 고려해야 함.

## 4. `query()` 내 비교 연산자 오용
- **현상**: `df.query("rank = 4")` 실행 시 문법 에러 발생.
- **원인**: `query` 문자열 내부에서도 비교 연산은 `=`이 아닌 `==`를 사용해야 함.
- **해결**: `df.query("rank == 4")`로 수정.
- **교훈**: 판다스 내부의 문자열 기반 엔진도 표준 파이썬 연산 규칙을 따름.

**기타 요약:**
- **`avg` vs `mean`**: 통계학적 정밀도(산술 평균)를 강조하기 위해 과학 라이브러리는 `mean`을 선호함.
- **필터링 vs 그룹화**: 성능과 논리적 간결함을 위해 **필터링을 먼저** 수행한 뒤 그룹화(`groupby`)하는 것이 유리함.
