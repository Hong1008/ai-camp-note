# [Troubleshooting] 2026-05-13 실습 리포트

## 1. 개요
- **일자**: 2026-05-13
- **학습 주제**: CNN 심화 및 컬러 이미지 분류 (CIFAR-10)
- **주요 목표**:
    - 컬러 이미지(RGB)의 CNN 처리 및 차원 관리(CHW) 이해
    - Data Augmentation 및 GAP를 통한 모델 일반화 성능 향상
    - 데이터 규모(Scaling)와 학습 효율성 사이의 최적점 탐색

## 2. 발생 이슈 및 해결 과정

### [Issue #1] 깊은 신경망에서의 과적합 및 성능 정체
- **증상**: 초기 CNN(58.5%) 및 심화 모델(65%)에서 Train/Val Loss 간격이 벌어지며 성능이 정체됨.
- **원인 분석**: 파라미터 수 급증으로 인한 오버피팅 및 컬러 이미지의 복잡한 특징 추출 한계.
- **해결 방안**: 
    - **Data Augmentation**: 무작위 반전 및 회전을 적용하여 데이터 다양성 확보.
    - **Global Average Pooling(GAP)**: FC 층의 파라미터를 획기적으로 줄여 일반화 성능 강화.
    - **Learning Rate Scheduler**: 정체기에서 LR을 낮춰 세밀한 최적화 유도.
- **결과**: 정확도 73.4% 달성 및 손실 곡선 안정화.

### [Issue #2] 데이터 규모와 학습 속도의 트레이드오프
- **증상**: 전체 데이터(50,000장) 학습 시 에포크당 소요 시간이 너무 길어 실험 효율 저하.
- **원인 분석**: 하드웨어 리소스와 모델 복잡도 대비 과도한 데이터 양.
- **해결 방안**: 데이터 샘플링을 통해 20,000장의 최적 데이터셋 구성(Stratified Split 적용).
- **결과**: 10,000장(73.4%) 대비 **81.5%**로 정확도 급상승하며 실무적 타협점 도달.

## 3. 핵심 학습 요약
- **데이터 구조**: NumPy(HWC)와 PyTorch(CHW)의 차원 차이를 이해하고 `transpose`를 통한 올바른 전처리가 필수적임.
- **규제 기법**: `BatchNorm2d`와 `Dropout`의 역할을 구분하고, 특히 GAP가 과적합 방지에 얼마나 강력한지 검증함.
- **점진적 고도화**: Baseline → 깊은 모델 → 규제/증강 → 데이터 스케일링으로 이어지는 고도화 단계의 중요성 체득.

---

## 4. 시니어 멘토 가이드 (Antigravity)
- **Production-Level Point**: 실제 현업에서는 모델 아키텍처 튜닝보다 **데이터 증강 전략과 규모 최적화**가 성능 향상에 더 빠르고 확실한 기여를 한다. 80% 이상의 정확도는 단순 수치 이상의 '모델 신뢰도'를 의미한다.
- **Analytical Insight**: Confusion Matrix를 통해 `Cat` vs `Dog`와 같은 **Semantic Confusion**을 파악하는 능력이 모델 개선의 시작이다. 단순히 점수만 보는 것이 아니라 어떤 데이터를 왜 틀리는지 분석하는 습관이 중요하다.
