# [Antigravity] 2026-05-12 트러블슈팅 리포트

오늘 CNN 실습 및 MNIST 고도화 과정에서 발생한 기술적 이슈와 해결 과정을 기록합니다.

---

## 1. CNN 채널 불일치 에러 (Channel Mismatch)
- **현상**: `RuntimeError: Given groups=1, weight of size [64, 1, 3, 3], expected input[64, 32, 28, 28] to have 1 channels, but got 32 channels instead.`
- **원인**: 두 번째 `nn.Conv2d` 레이어의 `in_channels`를 이전 레이어의 출력 채널 수(32)가 아닌 흑백 이미지 채널 수(1)로 잘못 설정함.
- **해결**: 두 번째 `nn.Conv2d`의 `in_channels`를 `32`로 수정하여 이전 레이어의 출력을 정상적으로 받도록 함.
- **교훈**: CNN을 깊게 쌓을 때, 현재 레이어의 입력 채널은 반드시 이전 레이어의 출력 채널과 일치해야 함을 인지함.

---

## 2. Linear 레이어 입력 차원 계산 오류
- **현상**: `Flatten` 이후 `nn.Linear`에서 `size mismatch` 에러 발생.
- **원인**: 레이어가 깊어지면서 이미지 크기가 28 -> 14 -> 7로 줄어든 것과 필터 개수가 늘어난 것을 반영하지 못하고 이전 모델의 숫자(`8*14*14`)를 그대로 사용함.
- **해결**: `Linear` 입력 차원을 `64(채널) * 7(가로) * 7(세로) = 3136`으로 수정함.
- **교훈**: `nn.MaxPool2d`가 이미지 크기에 미치는 영향($1/2$)을 고려하여 최종 `Flatten` 결과값을 산출해야 함. (헷갈릴 땐 `forward` 내에서 `x.shape` 출력을 활용할 것)

---

## 3. ROI 추출 시 종횡비(Aspect Ratio) 왜곡
- **현상**: 숫자 '1'과 같이 좁은 이미지를 $28 \times 28$로 리사이즈했을 때 가로로 비정상적으로 늘어나는 현상 발생. 이로 인해 모델이 '8' 등으로 오인식함.
- **원인**: `cv2.resize` 시 원본의 가로세로 비율을 무시하고 강제로 정사각형으로 늘림.
- **해결**: 원본의 비율을 유지하며 가장 큰 변을 기준으로 정사각형 캔버스(`np.zeros`)를 만든 뒤, 그 중앙에 숫자를 배치(Padding)하고 리사이즈함.
- **교훈**: 이미지 데이터 전처리 시 모델의 학습 데이터(MNIST)와 유사한 형태(중앙 배치, 여백 유지)를 갖추는 것이 인식률에 결정적임.

---

## 4. 학습 곡선에서의 검증 손실(Validation Loss) 반등
- **현상**: 학습 후반부(9~10 에포크)에서 `train_loss`는 줄어드나 `val_loss`가 다시 상승하는 현상.
- **원인**: 학습률(`lr=0.01`)이 다소 높고, 학습 데이터에 모델이 너무 특화되는 **과적합(Overfitting)** 발생.
- **해결**: 학습률을 `0.001`로 낮추고 에포크를 조절하여 최적의 가중치를 얻도록 개선함.
- **교훈**: 정확도가 높더라도 `val_loss`의 추이를 살펴 실제 일반화 성능을 항상 의심해야 함.
