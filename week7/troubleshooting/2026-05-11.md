# 2026-05-11 실습 리포트

## 📝 오늘 학습한 핵심 내용
- **딥러닝 고도화**: MLP 구조 설계, Dropout, Batch Normalization 적용을 통한 모델 안정화.
- **데이터 변환**: 타겟 데이터의 로그 변환(Log1p)을 통한 회귀 모델 성능 극대화(MAPE 4.5%).
- **이진 분류 마스터**: Sigmoid 활성화 함수의 역할 이해 및 분류 모델의 올바른 평가지표(F1-score, Recall) 학습.
- **불균형 데이터 대응**: 이직 예측(Attrition) 데이터의 클래스 불균형 문제를 `pos_weight`와 `BCEWithLogitsLoss`로 해결.

## 🛠️ 발생한 문제 및 해결 과정 (Troubleshooting)

### 1. 회귀 모델의 과적합(Overfitting) 발생
- **현상**: Epoch가 지날수록 Train Loss는 감소하나 Val Loss가 U자 형태로 다시 상승함.
- **원인**: 모델이 학습 데이터의 노이즈까지 외우기 시작하여 일반화 성능이 하락함.
- **해결**: 
    - `nn.Dropout(0.2)` 추가로 뉴런 의존성 분산.
    - `weight_decay=0.001` (L2 규제) 적용으로 가중치 크기 억제.
    - `StepLR` 스케줄러를 사용하여 학습 후반부 정밀도 향상.

### 2. 불균형 데이터의 '정확도 역설(Accuracy Paradox)'
- **현상**: 이직 예측 모델의 정확도가 90%를 넘지만, 정작 이직할 사람(1)의 재현율(Recall)은 0.47로 매우 낮음.
- **원인**: 다수 클래스(이직 안 함)의 비중이 압도적으로 많아 모델이 편향됨.
- **해결**: `pos_weight`를 계산하여 `BCEWithLogitsLoss`에 전달. Recall을 0.64까지 끌어올려 비즈니스 가치가 높은 모델로 개선.

### 3. 옵티마이저 미갱신으로 인한 학습 불능
- **현상**: 모델 구조 변경 후 학습을 돌렸으나 정확도가 14%로 급락하고 전혀 학습되지 않음.
- **원인**: 새로운 모델 객체(`model1`)를 생성했으나, 옵티마이저는 여전히 이전 모델의 가중치 주소를 가리키고 있었음.
- **해결**: 모델 정의 직후 `optimizer = Adam(model1.parameters(), ...)`를 재선언하여 동기화함.
